# LLM Tool Calling

## What is LLM Tool Calling?

LLM Tool Calling is the process where an **LLM decides which tool to use** based on the user's request.

The LLM **does not execute the tool**.

It only decides **which tool should be called** and **what arguments to pass**.

---

# Why is Tool Calling Needed?

Without Tool Calling:

- LLM can only generate text.
- Cannot access external systems.
- Cannot perform real-world actions.

With Tool Calling:

- Search information
- Send emails
- Access databases
- Use APIs
- Perform calculations

---

# bind_tools()

```python
llm.bind_tools(tools)
```

## Purpose

Makes the LLM aware of all available tools.

Without `bind_tools()`, the LLM cannot use MCP tools.

---

# What Does bind_tools() Do?

It gives the LLM information about:

- Tool name
- Description
- Parameters
- Parameter types

Example:

```
Tool:
add

Description:
Adds two numbers.

Parameters:
a
b
```

The LLM now knows how to use it.

---

# How Does the LLM Choose a Tool?

The LLM uses:

- User request
- Tool name
- Tool description
- Required parameters

It does **not** choose tools randomly.

---

# Example

Available Tools:

```
add()

subtract()

multiply()
```

User:

```
Add 15 and 25
```

LLM selects:

```
add()
```

---

User:

```
Subtract 10 from 30
```

LLM selects:

```
subtract()
```

---

# Does the LLM Execute the Tool?

❌ No

The LLM only decides.

Execution is performed by the MCP Server.

---

# JSON Generated by LLM

Example:

```json
{
  "tool": "add",
  "arguments": {
    "a": 15,
    "b": 25
  }
}
```

This request is sent to the MCP Client.

---

# Responsibilities

| Component | Responsibility |
|-----------|----------------|
| LLM | Chooses tool |
| MCP Client | Sends request |
| MCP Server | Executes tool |
| External API | Performs actual work |

# Standard vs Changeable

## Standard

- Tool calling process
- bind_tools()
- JSON-RPC communication
- MCP protocol

---

## Changeable

- LLM (GPT, Claude, Gemini)
- Tool names
- Tool descriptions
- Parameters
- Business logic

---

# Common Mistakes

❌ Thinking `bind_tools()` executes tools.

Reality:

It only informs the LLM about available tools.

---

❌ Thinking the LLM executes Python code.

Reality:

The MCP Server executes the code.

---

❌ Calling tools before binding.

Result:

The LLM doesn't know the available tools.
